Deployment Workflow

Deploying to bapX is designed to be invisible. Our GitHub-native workflow handles everything from provisioning to global edge distribution.

  1. Connect your GitHub repository in the bapX Console.
  2. Configure your bapx.json manifest file.
  3. Push to main. Your project is live at bapx.[name].in in seconds.
// bapx.json
{
  "name": "my-awesome-app",
  "framework": "nextjs",
  "storage": { "hub": "primary", "region": "us-east" },
  "compute": { "runtime": "wasm-edge" }
}

Sovereign Intelligence (AI Hub)

The bapX AI Hub (ai.bapx.in) is a reasoning and search engine powered by Open WebUI and serving local models via llama.cpp. It is directly wired to your central bapX database for unified authentication and storage.

Core AI Capabilities:

  • Web Search Loop: Integrated search providers (SearXNG, Google) for real-time data retrieval.
  • Reasoning Loop: Powered by Qwen2.5-Coder-7B for logical, step-by-step problem solving.
  • Lifetime Memory (RAG): Direct Postgres integration to query your user's specific lifetime data stored in bapXdb.

The Model Engine

We leverage llama.cpp to serve models locally on your infrastructure, ensuring data sovereignty. The default reasoning model is Qwen2.5-Coder-7B-Q8_0.

# Serve the model via bapX-CLI
./ai/bin/bapx-cli start

Storage as Compute (Deep Dive)

Traditional architectures separate data (Databases) from execution (Serverless/VPS). This introduces "data gravity" issues and latency. bapX eliminates this gap by allowing code to execute directly within the storage engine.

Key Technical Advantages:

  • Zero Data Transit: Data never leaves the secure memory space of the storage hub during processing.
  • Byte-Perfect Precision: Identities are mapped directly to data fragments, ensuring sovereign security.
  • Linear Scalability: As you add storage capacity, your compute power scales proportionally.
// Native Rust function running on Storage Hub
#[bapx_compute]
pub fn process_order(order: Order) -> Result {
    let inventory = storage::get_ref("inventory");
    if inventory.check(&order) {
        order.mark_processed();
        return Ok(OrderResult::Success);
    }
    Err(OrderError::OutOfStock)
}
