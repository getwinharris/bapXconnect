v1.8.5 January 3, 2026

Database Completeness & Systemic Testing

Strengthening the core with unified Postgres storage, bit-perfect identity tensors, and autonomous system verification bots.

Core Infrastructure

  • Unified Postgres: Fully wired Open WebUI and Core Runtime to a shared Postgres backend for seamless auth and memory persistence.
  • Storage-as-Compute: Implemented the BAPX Identity Tensor (x8dtensor) for bit-perfect data reduction and restoration.

  • Memory Mapping: Successfully mapped personal memory and storage from Open WebUI to the unified user database.
  • Enhanced Search: Initiated Grok-style dynamic search loops with cross-session context and knowledge base integration.
v1.8.4 January 3, 2026

Qwen2.5-Coder & llama.cpp Integration

Bringing elite coding performance to the sovereign intelligence layer with Qwen2.5-Coder-7B and high-efficiency llama.cpp orchestration.

Intelligence Evolution

  • Model Update: Integrated Qwen2.5-Coder-7B-Q8_0 as the primary engine for ai.bapx.in.
  • Performance: Switched to llama.cpp for serving models, significantly reducing latency and memory footprint.
  • Unified Auth: Single username and password across bapx.in and ai.bapx.in. Sign up once, use everywhere.
  • Product Switcher: Added a native sidebar menu for seamless switching between bapXdb and AI Hub.
  • Tier Refinement: Free users now receive Lifetime Memory Context but model access is reserved for Pilot/Sovereign plans.

The Open WebUI & API Integration

Deprecating legacy automation tools in favor of a model-first intelligence layer powered by Open WebUI and native Model APIs.
